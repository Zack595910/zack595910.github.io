五、任务引擎详解

5.1 任务文件格式（.ini）

[Meta] ← 任务元信息 id=quest_001 ← 任务唯一ID name=任务名称 ← 显示名称 type=main ← main=主线, side=支线 npc_name=NPC名字 ← 关联NPC npc_id=npc_001 ← NPC唯一标识 [Prerequisites] ← 前置条件（可选） level_min=1 ← 最低等级 quests_before=quest_000 ← 前置任务ID [Objectives] ← 任务目标 obj_0_type=collect ← 类型: collect/reach/defeat/talk obj_0_item=herb ← 目标物品 obj_0_amount=5 ← 目标数量 obj_0_text=采集草药 ← 显示文字 [Rewards] ← 任务奖励 coins=50 ← 金币 exp=30 ← 经验 items=medicine:2,scroll:1 ← 物品:数量 [Dialogs] ← 对话节点配置 start=node_start ← 起始对话 progress=node_progress ← 进行中对话 complete=node_complete ← 完成对话

八、数据流

Run.bat → 启动 Python Launcher\python.exe
→ 执行 main.py（初始化KSDMain类）
→ 加载 ResourceManager（万能文件处理器）
→ user_config.check_login()（用户登录/注册）
→ quest_engine._load_persistence()（恢复任务进度）
→ game_loop.setup_canvas()（构建画布/WASD绑定）
→ 构建UI + 启动所有子系统（天气/小地图/NPC检测/联机）
→ 用户操作 → 菜单事件 → 调用各模块方法
→ 靠近NPC → 自动检测（60px范围）→ 弹出对话界面
→ 模块加载 → exec() 动态执行
→ 数据读写 → configparser INI / json JSON

十一、常见问题 FAQ

Q: 游戏数据存储在哪里？
所有数据在 C:\Date\Snxth.\KSD Run Files\，用户/背包/任务进度/成就/排行榜均为独立文件。
Q: 任务系统怎么使用？
将 .ini 放入 External Modules\Quests\，靠近带 ! 的NPC接受，完成目标后靠近带 ? 的NPC交付。
Q: 如何局域网联机？
游戏选项 → 局域网联机，一台创建主机，另一台输入IP加入。确保两台电脑在同一局域网。
Q: 如何成为管理员？
导入密钥文件或输入 SNXTH-ADMIN-2026，验证通过后提升为管理员。
Q: 如何恢复出厂设置？
运行 KSD_reset.py，支持双验证，建议先备份再重置。
