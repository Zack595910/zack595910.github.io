📖 项目简介

KSD 是一个基于 Python Tkinter 构建的模块化冒险游戏框架。自 Core 1.0 起经历七个架构版本迭代，现已发展为功能完善的游戏平台。

支持用户系统、背包系统、密钥管理、主题切换、Shell调试面板、模块动态加载等丰富功能。项目由 闪讯工作社 Snxth. 开发维护，使用 Python 标准库构建，无需第三方依赖。

数据存储在 C:\Date\Snxth.\KSD Run Files\，使用 INI 和 JSON 格式持久化。内置向导式任务编辑器，支持第三方模块导入和动态加载。

✨ 核心功能

🎮

经典玩法

WASD控制角色移动，收集金色经验球升级，每球+1金币。等级无上限，经验球每3秒自动补充，画布始终保持5个球。升级通知、经验进度条实时显示。

👤

用户系统

注册/登录/记住密码/退出登录。密码SHA256加密存储。管理员权限分离。每个用户独立存档（等级/背包/金币），支持切换账户。

🎒

背包系统

10种物品：木材/石头/铁锭/金块/食物/水/药品/工具/宝石/卷轴。容量50格。支持购买/出售（半价回收）/清空。管理员免费获取。

🔑

密钥系统

6种验证方式：明文密钥/SHA256哈希/Base64编码/JSON结构/主密钥/加密双重验证。支持密钥冻结、使用日志（保留200条）、密钥生成（JSON/加密/主密钥格式）。

🎨

主题切换

三种主题：经典Windows（SystemButtonFace）、深色模式（深灰背景浅色文字）、浅色模式（浅灰背景深色文字）。实时切换，影响画布/信息栏/Shell面板。

💻

Shell面板

底部终端风格日志窗口，深色背景绿色文字。6种彩色标签：SYSTEM(橙)/ERROR(红)/WARNING(黄)/INFO(蓝)/SUCCESS(绿)/CODE(灰)。每条日志带时间戳，支持Clear清空。

📦

模块管理

内置模块管理器（需管理员权限）。扫描External Modules文件夹，显示已安装模块列表。支持导入新模块（复制到文件夹）、加载选中模块（exec执行）、卸载模块（删除文件）。模块信息持久化在installed_modules.json。

📝

任务编辑器

4步向导式创建INI任务文件：基本信息→显示内容→奖励设置→生成文件。步骤指示器显示进度（蓝色当前/绿色完成/灰色未到）。生成文件带注释头，支持预览后保存并导入测试。

📷

截图系统

F12一键截图，保存为PostScript格式。文件名包含时间戳和序号，存入Screenshots文件夹。截图计数自动递增。

💾

备份恢复

完整数据备份（复制整个data文件夹到指定位置）和恢复功能。支持JSON格式（完整结构化数据）和CSV格式（表格形式）导出。导出后自动打开文件夹。

📋 操作指南

🎮 游戏操作

1
使用 W A S D 或 ↑ ↓ ← → 方向键控制角色在画布区域移动
2
移动角色触碰画布中的金色圆形经验球即可收集，每球获得 1经验 + 1金币
3
经验值达到当前等级所需上限（等级×10）后自动升级，经验归零重新累计，等级无上限
4
画布中始终维持最多5个经验球，每3秒自动补充一个
5
按 F12 键截取当前画布画面，保存到 Screenshots 文件夹

📦 模块导入步骤

1
打开菜单栏 控制面板 → 配置与功能（需要管理员权限，首次使用需导入密钥）
2
点击 "导入新模块" 按钮，在文件选择框中选择 .py 或 .ini 格式的模块文件
3
模块自动复制到 External Modules 文件夹，并出现在已安装模块列表中
4
在列表中 选中模块，点击 "加载选中模块" 即可运行该模块
5
模块运行中可通过 游戏选项 → 退出当前任务 返回主界面，或通过模块管理器 "卸载选中" 删除
💡 提示：模块信息保存在 Cache\installed_modules.json，每次打开管理器会自动扫描同步文件夹。模块通过全局变量 game 访问主程序功能。

🔑 获取管理员权限

1
打开 控制面板 → 配置与功能 → 密钥管理 → 导入密钥
2
选择密钥文件或输入密钥内容（如 SNXTH-ADMIN-2024）
3
验证通过后立即提升为管理员，信息栏显示 [管理员] 标识
4
管理员可在密钥管理面板生成新密钥（JSON/加密/主密钥格式）分享给其他用户

🏗️ 技术架构

📁 完整文件结构

C:\Date\Snxth.\
├─ KSD main\ ← 主程序目录
├─ main.py ← 主界面+菜单+关于+模块管理器+向导编辑器+通知系统+主题系统
├─ Core Modules\ ← 核心功能模块目录
│ ├─ user_config.py ← 用户系统(注册/登录/记住密码/背包/金币/主题/存档/用户管理)
│ ├─ game_loop.py ← 游戏主循环(WASD移动/经验球生成/碰撞检测/升级判定/初始任务)
│ ├─ file_manager.py ← 文件管理(数据目录初始化/备份恢复/JSON+CSV导出/存档加载)
│ └─ screenshot.py ← 截图功能(F12触发/PostScript格式/自动计数)
└─ Python Launcher\ ← Thonny便携版(Python运行环境，免安装)
├─ KSD Run Files\ ← 运行时数据目录
├─ Cache\ ← 配置缓存(主题配置theme_config.ini/模块列表installed_modules.json)
├─ Keys\ ← 密钥文件存储(JSON/KEY格式)
├─ Users\ ← 用户存档(users.ini/login_config.ini/各用户save_/backpack_/coins_的INI文件)
├─ External Modules\ ← 第三方模块(.py/.ini任务文件，模块管理器扫描此目录)
└─ Screenshots\ ← 截图文件(.ps格式，文件名含时间戳和序号)
└─ Run.bat ← 启动脚本(调用Python Launcher\python.exe执行main.py)

🔄 数据流

Run.bat → 启动 Python Launcher\python.exe
→ 执行 main.py（初始化KSDMain类）
→ 创建数据目录 → 初始化各模块实例
→ 调用 user_config.check_login()（用户登录/注册）
→ 调用 game_loop.setup_canvas()（构建画布/WASD绑定）
→ 构建UI（菜单栏/信息栏/Shell面板/通知系统/主题应用）
→ 启动游戏主循环（移动检测/碰撞检测/升级判定/经验球生成）
→ 用户操作 → 菜单事件 → 调用各模块方法
→ 模块加载 → exec() 动态执行第三方代码 → 通过全局变量 game 访问主程序
→ 数据读写 → configparser 操作 INI 文件 / json 操作 JSON 文件

❓ 常见问题 FAQ

Q: 游戏数据存储在哪里？
所有数据存储在 C:\Date\Snxth.\KSD Run Files\ 目录下。用户信息在 Users\users.ini，各用户背包/金币/存档为独立INI文件，模块信息在 Cache\installed_modules.json，密钥在 Keys\ 文件夹。
Q: 忘记密码怎么办？
密码使用SHA256加密存储，无法逆向解密。管理员可以在 用户选项 中删除该用户重新注册。或者直接编辑 Users\users.ini 文件，删除对应用户的整个区块（从 [用户名] 到下一个 [ 之前的内容）。
Q: 如何安装第三方模块？
打开 控制面板 → 配置与功能（需管理员权限，首次使用需导入密钥如 SNXTH-ADMIN-2024），点击 "导入新模块"，选择 .py 或 .ini 文件。模块会自动复制到 External Modules 文件夹并出现在列表中。
Q: 模块加载后无法正常使用怎么办？
请确认模块文件是 KSD Core 7.0 兼容格式。模块需通过全局变量 game 访问主程序功能（如 game.add_coins()、game.shell_log()、game.user.backpack 等）。如果模块是旧版（Core 5.0及以前）格式，需手动改写为Core 7.0兼容格式。
Q: 如何备份我的游戏进度？
方法1：打开菜单 文件 → 备份数据，选择目标文件夹即可完整备份整个KSD Run Files。方法2：手动复制 C:\Date\Snxth.\KSD Run Files\ 整个文件夹到安全位置。恢复时使用 文件 → 恢复数据 或手动复制回原位。
Q: 如何成为管理员？
在 控制面板 → 配置与功能 → 密钥管理 中导入有效密钥。内置密钥包括：SNXTH-ADMIN-2024（明文）、SNXTH-ADMIN-2024-MASTER-KEY-FLASH-STUDIO（主密钥）、FLASH-STUDIO-MASTER。管理员可以生成新密钥分享给其他用户。
Q: 游戏窗口太小/太大怎么办？
KSD窗口固定为800×650像素，如需调整可在 main.py 中找到 self.root.geometry("800x650") 修改数值。画布为800×450，Shell面板高度100px。修改后需同步调整画布创建参数。
