NPC头顶自动显示任务状态标记：黄色感叹号 ! 表示可接取新任务，蓝色问号 ? 表示任务已完成可交付。标记为圆形背景+文字的组合图形，与NPC名字文本分离绘制。玩家靠近NPC（60像素范围内）时自动触发交互，弹出对应的任务对话框。任务状态变更时标记动态刷新。

5.4 任务持久化（JSON双文件）

文件存储内容更新时机
active_quests.json进行中任务的完整进度（目标类型、物品名称、当前进度、目标数量、显示文字）每次进度变更自动写入
completed_quests.json已完成任务的永久记录（任务名称、完成时间戳、奖励明细）任务交付时写入

启动KSD时自动调用加载函数恢复数据，关闭时无需手动保存——所有进度实时持久化，断电不丢失。

5.5 自动恢复任务追踪

启动时自动检测活跃任务，恢复右上角浮动追踪面板。追踪面板以白色背景、黑色文字显示在窗口左上角，列出当前所有活跃任务的目标进度。可交付的任务会额外显示"✅ 可交付!"提示。NPC标记同步刷新。

5.6 任务日志面板

控制面板新增"任务日志"按钮，打开500×500独立窗口，分两个区域展示：

• 进行中区域 — 每个任务的目标进度+完成状态+可交付提示

• 已完成区域 — 任务名称+完成时间+奖励明细（金币/经验/物品）

窗口采用白色经典Windows风格，与整体界面统一。

5.7 任务文件自动清理

任务完成交付后，对应的 .ini 文件自动从 External Modules\Quests\ 目录删除，避免重复加载和文件堆积。完成记录永久保留在 completed_quests.json 中。

七、操作指南

🎮 游戏操作

1
使用 W A S D 或 ↑ ↓ ← → 方向键控制角色在画布区域移动
2
移动角色触碰画布中的金色圆形经验球即可收集，每球获得 1经验 + 1金币
3
经验值达到当前等级所需上限（等级×10）后自动升级，经验归零重新累计，等级无上限

画布中始终维持最多5个经验球，每3秒自动补充一个
5
按 F12 键截取当前画布画面，保存到 Screenshots 文件夹

📜 任务系统使用

1
将任务 .ini 文件放入 External Modules\Quests\ 目录，KSD会自动扫描并识别
2
游戏中靠近带 ! 标记的NPC，弹出对话框接受任务
3
完成任务目标后，靠近带 ? 标记的NPC交付任务领取奖励
4
打开 控制面板 → 任务日志 查看进行中和已完成的任务详情
5
任务完成后，对应的.ini文件会自动从Quests目录删除，但完成记录永久保留
💡 任务进度自动保存到 Cache/active_quests.json，关闭KSD后重新打开会自动恢复未完成任务的追踪状态。

Run.bat → 启动 Python Launcher\python.exe

→ 调用 user_config.check_login()（用户登录/注册）
→ 调用 quest_engine._load_persistence()（恢复任务进度）

→ 构建UI（菜单栏/信息栏/Shell面板/通知系统/主题应用）
→ 启动游戏主循环 + NPC检测循环（300ms间隔）+ 任务追踪恢复
→ 用户操作 → 菜单事件 → 调用各模块方法
→ 靠近NPC → 自动检测（60px范围）→ 弹出对话界面
→ 模块加载 → exec() 动态执行第三方代码 → 通过全局变量 game 访问主程序
→ 数据读写 → configparser 操作 INI 文件 / json 操作 JSON 文件

十一、常见问题 FAQ

Q: 游戏数据存储在哪里？
所有数据存储在 C:\Date\Snxth.\KSD Run Files\ 目录下。用户信息在 Users\users.ini，各用户背包/金币/存档为独立INI文件，模块信息在 Cache\installed_modules.json，任务进度在 Cache\active_quests.json 和 Cache\completed_quests.json，密钥在 Keys\ 文件夹。
Q: 任务系统怎么使用？
将任务 .ini 文件放入 External Modules\Quests\ 目录，KSD启动时自动扫描。游戏中靠近带 ! 标记的NPC接受任务，完成任务后靠近带 ? 标记的NPC交付。可在 控制面板 → 任务日志 查看所有任务状态。任务进度自动保存，关闭后重新打开自动恢复。
Q: 任务编辑器怎么用？
方式1：使用KSD内置的 文件 → 向导式任务编辑器（4步引导创建简单任务）。方式2：运行独立的 ksdeditor.py 图形化编辑器，支持对话节点编辑、地图编辑、Python代码编辑、导出为.ini文件或保存为.ksd项目文件。
Q: 忘记密码怎么办？
密码使用SHA256加密存储，无法逆向解密。管理员可以在 用户选项 中删除该用户重新注册。或者直接编辑 Users\users.ini 文件，删除对应用户的整个区块（从 [用户名] 到下一个 [ 之前的内容）。
Q: 如何安装第三方模块？
打开 控制面板 → 配置与功能（需管理员权限，首次使用需导入密钥如 SNXTH-ADMIN-2026），点击 "导入新文件"，选择 .py 或 .ini 文件。模块会自动复制到 External Modules 文件夹并出现在列表中。
Q: 模块加载后无法正常使用怎么办？
请确认模块文件是 KSD Core 8.0 兼容格式。模块需通过全局变量 game 访问主程序功能（如 game.add_coins()、game.shell_log()、game.user.backpack 等）。如果模块是旧版格式，需手动改写为Core 8.0兼容格式。
Q: 如何备份我的游戏进度？
方法1：打开菜单 文件 → 备份数据，选择目标文件夹即可完整备份整个KSD Run Files。方法2：运行 KSD_maintainer.py 运维小助手，点击"创建备份"。方法3：手动复制 C:\Date\Snxth.\KSD Run Files\ 整个文件夹到安全位置。恢复时使用对应方法还原。
Q: 如何成为管理员？
在 控制面板 → 配置与功能 → 密钥管理 中导入有效密钥。内置密钥包括：SNXTH-ADMIN-2026（明文）、SNXTH-ADMIN-2026-MASTER-KEY-FLASH-STUDIO（主密钥）、FLASH-STUDIO-MASTER。管理员可以生成新密钥分享给其他用户。
Q: 如何恢复出厂设置？
运行 KSD_reset.py 独立工具（可放在任意位置）。支持两种验证方式：输入"确认重置"或导入管理员密钥文件。建议先点击"先备份再重置"安全操作。清除范围包括所有用户数据、任务进度、存档、密钥、截图，保留主程序和外部模块。
Q: 游戏窗口太小/太大怎么办？
KSD窗口固定为800×650像素，如需调整可在 main.py 中找到 self.root.geometry("800x650") 修改数值。画布为800×450，Shell面板高度100px。修改后需同步调整画布创建参数。
